# ADR-001: サーバーレスアーキテクチャの採用

## Status

Accepted

## Context

画像リサイズ API の Compute 基盤を選定する必要がある。
個人ポートフォリオとして以下の制約がある:

- 月額コストを $5 以下に抑えたい
- 運用負荷を最小化したい（個人で運用するため）
- トラフィックは不定期で、アイドル時間が長い

## Options

### Option A: AWS Lambda (サーバーレス)

- 実行した分だけ課金（アイドル時コスト $0）
- 月100万リクエスト無料枠
- OS/ランタイムのパッチ不要
- コールドスタートあり（~1秒）

### Option B: Amazon ECS Fargate

- コンテナベースで柔軟な構成が可能
- 最小構成でも ~$15/月のコスト
- コールドスタートなし
- コンテナイメージの管理が必要

### Option C: Amazon EC2

- 最も自由度が高い
- t3.micro で ~$5/月（ただし常時稼働）
- OS/セキュリティパッチの管理が必要
- スケーリングは手動 or ASG 設定が必要

## Decision

**Option A: AWS Lambda** を採用する。

## Reasoning

1. **コスト**: 画像リサイズは断続的な処理であり、常時稼働は不要。Lambda は実行時のみ課金されるため、個人利用（月1,000リクエスト）では実質無料で運用できる

2. **運用負荷**: サーバー管理が不要で、デプロイも ZIP アップロードのみ。個人運用に最適

3. **スケーリング**: トラフィック増加時も自動スケールし、設定不要

4. **トレードオフ**: コールドスタート（~1秒）は画像リサイズのユースケースでは許容範囲。リアルタイム処理が必要な場合は Provisioned Concurrency で対応可能だが、コスト増となるため現時点では不採用
